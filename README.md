## 🎮 프로젝트 소개

**우결 사다리게임**은 우리 결혼했어요(We Got Married) 스타일의 남녀 커플 매칭 사다리타기 웹 앱입니다.

방송 스트리밍이나 화면공유 환경에서 **주최자 혼자 운영**할 수 있도록 설계되었으며,  
화면에 전혀 티가 나지 않게 특정 참가자를 결과에서 제외하는 **비밀 확률 0% 기능**을 제공합니다.

<br/>

## ✨ 주요 기능

### 🎯 사다리 게임
- 전통적인 한국식 사다리타기 — 모든 남자가 열에 배치, 위에서 아래로 내려옴
- 상단 이름 칩 클릭 → 해당 인원의 경로만 애니메이션 (150ms/step)
- **전체 사다리 타기!** 버튼 → 남자 순서대로 순차 애니메이션
- 경로가 완성될 때마다 결과가 한 줄씩 아래에 등장
- **다시 돌리기** → 새 사다리 + 새 슬롯 배치로 완전 리셋

### 👫 성비 불균형 완전 지원
- **남자 > 여자**: 여자 수만큼 슬롯 배치 + 나머지 자리에 `💫 언젠가` 슬롯 랜덤 배치  
  → 해당 남자 결과: **To be continued...**
- **여자 > 남자**: 남자들이 모두 매칭된 후 남은 여자들이 결과 하단에 별도 표시  
  → 구분선 `아직 인연을 기다리는 중 💫` 아래에 **To be continued...** 로 표시

### 👥 참가자 관리
- 이름 + 성별(남/여) 입력, Enter 키 지원
- 이름 중복 방지
- 참가자 삭제
- 남자/여자 별도 컬럼 표시
- 실시간 인원 통계 (남 N명 💞 여 N명)

### 🔒 비밀 확률 0% 기능 (화면공유 완전 안전)
- 헤더 우측 **⚙️** 클릭 → 참가자 설정 모달 오픈
- 체크된 참가자는 **결과에서 항상 "To be continued..."** 처리 (당첨 불가)
- **참가자 목록에서는 아무 변화 없음** — 취소선, 색 변화 없이 완전히 티 안 남
- 제외 상태는 ⚙️ 모달 안에서만 확인 가능
- 제외된 남자는 사다리 위에 여전히 표시 (점선 테두리로만 구분)
- 모달 이름 "참가자 설정"은 중립적 → 화면공유 중 열려 있어도 자연스러움

### 🎨 파스텔 플랫 디자인
- 그라데이션 없는 단색 파스텔 팔레트 전체 적용
- 남자: 파스텔 스카이블루 / 여자: 파스텔 로즈핑크
- 모든 색상에 대비 높은 어두운 텍스트 색상 매칭 (접근성 고려)
- fadeUp / scaleIn / heartbeat / softPulse 마이크로 애니메이션

### 📱 완전 반응형
- 데스크탑 / 태블릿 / 모바일 모든 화면 최적화
- 모바일에서 남자/여자 목록이 세로로 자동 전환
- 사다리 캔버스는 참가자가 많을 때 가로 스크롤 대응
- 화면 너비에 따라 사다리 칸 너비 자동 조정 (62px ~ 90px)

- Node.js 18 이상

## 📖 사용 방법

### Step 1 — 참가자 추가
1. 성별 버튼(남자 / 여자)으로 성별 선택
2. 이름 입력 후 **Enter** 또는 **추가** 버튼 클릭
3. 원하는 만큼 반복 추가 (성비가 달라도 게임 가능)

### Step 2 — 비밀 확률 0% 설정 (선택, 화면공유 안전)
1. 헤더 오른쪽 **⚙️** 아이콘 클릭
2. 당첨되면 안 되는 참가자 체크
3. **확인** 클릭 → 참가자 목록에선 아무 변화 없음
4. 해당 참가자는 사다리에 표시되지만 결과는 항상 "To be continued..."

### Step 3 — 게임 시작
- 남자 1명 이상 + 여자 1명 이상이면 **🎉 사다리 게임 시작!** 버튼 활성화
- 버튼 클릭 → 게임 화면으로 이동

### Step 4 — 사다리 타기
- **이름 칩 클릭** → 해당 남자의 경로만 애니메이션으로 그려짐
- **🎲 전체 사다리 타기!** → 모든 남자 순서대로 자동 실행

### Step 5 — 결과 확인
- 경로 완성 시마다 커플이 한 줄씩 등장
- 여자가 부족하면 → 해당 남자는 **"To be continued..."**
- 여자가 남자보다 많으면 → 남은 여자들이 하단에 **"To be continued..."** 로 표시

### Step 6 — 다시 돌리기
- **🔄 다시 돌리기** → 새 사다리 + 새 슬롯 배치로 완전 재시작
- **← 참가자 수정** → 참가자 입력 화면으로 돌아가기

### 파스텔 컬러 팔레트 (단색, 그라데이션 없음)

| 변수 | 배경 | 텍스트 | 사용처 |
|------|------|--------|--------|
| `--p-rose` | `#FBCFE8` | `#9D174D` | 여자 칩·태그, 추가 버튼 |
| `--p-blue` | `#BAE6FD` | `#075985` | 남자 칩·태그 |
| `--p-purple` | `#DDD6FE` | `#4C1D95` | 시작 버튼, 확인 버튼 |
| `--p-mint` | `#A7F3D0` | `#065F46` | 다시 돌리기 버튼 |
| `--p-slate` | `#E2E8F0` | `#475569` | 빈 슬롯(언젠가) |

### 반응형 브레이크포인트

| 범위 | 주요 변화 |
|------|----------|
| `> 640px` | 기본 레이아웃 |
| `≤ 640px` | 패딩·폰트 축소, 부제목 숨김 |
| `≤ 480px` | 남자/여자 목록 세로 스택, 사다리 칸 72px |
| `≤ 360px` | 사다리 칸 최소 62px |

### 애니메이션

| 이름 | 동작 | 사용처 |
|------|------|--------|
| `fadeUp` | 아래→위 페이드인 | 화면 전환, 결과 항목 등장 |
| `scaleIn` | 작→큰 페이드인 | 모달, 결과 패널 |
| `heartbeat` | 두근거림 | 하트 이모티콘 |
| `softPulse` | 그림자 맥박 | 시작 버튼, 애니메이션 중 칩 |

## 🧩 컴포넌트 상세

### `App.jsx`
- 화면 상태 관리: `'setup'` ↔ `'game'`
- 참가자 배열 CRUD: `addParticipant`, `removeParticipant`, `toggleExclude`
- LadderGame에 남자/여자 전체 배열 전달 (제외 필터링은 LadderGame 내부 처리)

### `ParticipantSetup.jsx`

| 하위 | 역할 |
|------|------|
| `SecretModal` | ⚙️ 비밀 확률 0% 설정 모달 — 목록 화면에 티 안 남 |
| `GenderList` | 남자/여자 참가자 컬럼 렌더링 |

### `LadderGame.jsx`

| 순수 함수 | 역할 |
|-----------|------|
| `buildBridges(cols)` | 랜덤 가로 다리 생성 (행마다 인접 중복 방지) |
| `traverse(bridges, startCol)` | 한 열 사다리 타기 → 경로 포인트 배열 + 최종 열 반환 |
| `buildBottomSlots(females, cols)` | 여자 슬롯 + 빈 슬롯 Fisher-Yates 셔플 배치 |
| `shuffle(arr)` | Fisher-Yates 셔플 |
| `getColMetrics()` | 화면 너비 기반 열 너비·패딩 동적 반환 |
| `renderCanvas(...)` | Canvas 렌더링 — 레일, 다리, 글로우 경로 |

**결과 처리 규칙:**
- 제외된 남자 → 사다리 경로 정상 실행, 결과만 "To be continued..." 강제
- 남자 > 여자 → 빈 슬롯에 도착한 남자는 "To be continued..."
- 여자 > 남자 → 매칭 안 된 여자들 별도 섹션에 "To be continued..." 표시
